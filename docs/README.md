## 🎯 文档维护规范

根据项目管理规范：

1. ✅ **保留**: 核心使用文档和技术指南
2. ❌ **删除**: 临时修复记录、重构计划等开发中间文档
3. 🧹 **清理**: 定期清理自动生成的 .md 文件
4. 📦 **发布**: 对外发布时只保留必要的部署和使用说明

## 📝 添加新文档

当需要添加新的技术文档时：
1. 将文档放在此目录
2. 更新本 README 的文档列表
3. 确保文档有清晰的标题和目录结构
4. 避免与现有文档内容重复
