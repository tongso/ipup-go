# 项目开发规范总览

## 📁 项目目录结构规范

```
myproject/
├── cmd/                 # 应用启动入口
├── internal/            # 内部业务逻辑（私有包）
│   ├── app/            # API 暴露层（前端接口）
│   ├── config/         # 配置管理
│   ├── database/       # 数据库操作
│   ├── domain/         # 域名管理
│   ├── log/            # 日志系统
│   ├── monitor/        # 监控服务
│   ├── notify/         # 通知服务
│   └── provider/       # 第三方服务提供商
├── pkg/                # 公共包（可复用）
│   └── types/          # 类型定义
├── scripts/            # 工具和测试脚本
├── docs/               # 项目文档
├── frontend/           # 前端代码
├── main.go             # 程序入口
├── app.go              # 应用主结构
└── wails.json          # Wails 配置
```

## 📝 文件命名规范

- **测试脚本**：`test_*.go`, `verify_*.go`（小写 + 下划线）
- **文档文件**：`*.md`（大写 + 下划线或驼峰）
- **Go 源文件**：小写 + 下划线或驼峰
- **构建标签**：测试脚本必须添加 `//go:build ignore`

## 🏗️ 架构分层规范

### 依赖层级
```
main.go → app.go (API 层) → 业务模块 → database
```

### 模块职责
- **`internal/app/`**: 暴露给前端的 API 接口
- **`internal/*`**: 业务逻辑实现
- **`pkg/types`**: 公共类型定义
- **`scripts/`**: 独立运行的工具脚本

## 🔧 开发规范

### 1. Wails 绑定流程
```bash
# 修改 Go 方法后
1. 验证语法：get_problems
2. 重新生成绑定：wails generate module
3. 重启开发环境：wails dev
```

### 2. 类型同步
- Go 类型定义在 `pkg/types`
- 前端 TypeScript 类型必须与 Go结构同步
- 修改后立即更新前端类型

### 3. 数据库操作
- 所有查询必须显式处理 NULL 值
- 使用 COALESCE 或 IFNULL 转换 NULL
- 优先在 SQL 层面处理 NULL

### 4. 定时器管理
- 禁止硬编码刷新间隔
- 从配置中读取动态值
- 组件销毁时清理定时器
- 每个实例创建独立定时器

### 5. 用户反馈
- 禁止使用原生 alert()/confirm()
- 使用统一通知系统
- 支持成功/错误/信息/警告类型
- 非阻塞设计，自动消失

### 6. 组件通信
- 使用 CustomEvent 事件驱动
- 避免轮询，实时响应
- 事件触发后立即刷新数据

## 📋 代码质量

### 1. API 调用可观测性
- 所有外部 API 调用必须记录详细日志
- 包含：时间、域名、操作、参数、状态、错误
- 实时更新前端监控面板
- 支持按域名、时间、级别筛选

### 2. 状态展示
- 颜色标识（绿色成功/红色失败）
- 图标提示
- 点击查看详情
- 与定时任务联动

### 3. 手动操作支持
- 每个域名提供手动更新按钮
- 立即反馈执行结果
- 支持紧急更新
- 手动/自动共用同一逻辑

## 🔒 安全规范

- 不在生产环境暴露 dev server 端口
- 敏感配置（AccessKey）需要隐藏显示
- 防止 XSS 攻击（前端输入过滤）
- SQLite 数据库文件妥善保护

## 📊 性能优化

- IPv4/IPv6 并发获取
- 智能降级策略
- HTTP 超时控制
- 避免重复请求
- 缓存合理 TTL

## 🎯 最佳实践

1. **模块化**：职责单一，降低耦合
2. **接口化**：通过接口通信，便于替换
3. **配置化**：避免硬编码，支持动态调整
4. **可观测**：详细日志，实时监控
5. **容错性**：错误处理，降级策略
6. **可维护**：清晰注释，完整文档
